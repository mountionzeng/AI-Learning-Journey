## 第 7 章 小波变换与多分辨率处理

### 7.1 背景：为什么需要小波变换？

**傅里叶变换的局限性：**
- 只能告诉我们"有什么频率"
- 无法告诉我们"频率在什么时候出现"
- 适合分析平稳信号，不适合分析非平稳信号（包含突变）

**小波变换的优势：**
- 同时具有频率信息和位置信息
- 可以分析信号的局部特征
- 适合处理突变、边缘等信息

```plaintext
傅里叶变换：
只能看到"包含哪些频率"
         ↓
小波变换：
既知道"有什么频率"
又知道"在哪个位置"
```

### 7.2 小波的基本概念

**母小波函数 $\psi(t)$：**
- 有限持续时间的振荡波形
- 能量有限：$\int |\psi(t)|^2 dt < \infty$
- 满足容许条件：$\int \frac{|\Psi(\omega)|^2}{|\omega|} d\omega < \infty$

**基本性质：**
- 振荡性：有正有负，平均值为零
- 衰减性：在两端迅速衰减到零
- 有限持续时间

**常见小波基：**

| 小波名称 | 特点 | 应用 |
| --- | --- | --- |
| Haar | 最早、最简单，方波形状 | 教学、初步分析 |
| Daubechies (Db) | 消失矩高，紧支撑 | 通用应用 |
| Symlet | 对称性改善 | 图像处理 |
| Coiflet | 近似对称 | 信号去噪 |

### 7.3 连续小波变换（CWT）

**数学定义：**

$CWT_f(a,b) = \int_{-\infty}^{\infty} f(t) \psi_{a,b}^*(t) dt = \int f(t) \frac{1}{\sqrt{|a|}} \psi^* \left( \frac{t-b}{a} \right) dt$

**参数解释：**
- $a$：尺度参数（Scale），控制小波的伸缩
  - $a > 1$：小波伸展，低频分析
  - $a < 1$：小波压缩，高频分析
- $b$：平移参数（Translation），控制小波的位置

**直观理解：**
- 用不同尺度的小波去"扫描"信号
- 尺度 $a$ 小 → 压缩的小波 → 檢測高频细节
- 尺度 $a$ 大 → 伸展的小波 → 分析低频轮廓

### 7.4 离散小波变换（DWT）

**离散化策略：**
- 尺度：$a = 2^j$（二进制伸缩）
- 平移：$b = k \cdot 2^j$（二进制平移）

**二进制小波：**

$\psi_{j,k}(t) = 2^{-j/2} \psi(2^{-j}t - k)$

**二维离散小波变换分解：**

```plaintext
原始图像 (N×N)
    │
    ├─→ 水平低频 + 垂直低频 → LL (近似分量/低频)
    │
    ├─→ 水平高频 + 垂直低频 → HL (水平边缘)
    │
    ├─→ 水平低频 + 垂直高频 → LH (垂直边缘)
    │
    └─→ 水平高频 + 垂直高频 → HH (对角线边缘)

每次分解后，图像尺寸各减半
```

**分解示意：**

```plaintext
原始图像
┌─────┬─────┐
│  LL │  HL │
├─────┼─────┤
│  LH │  HH │
└─────┴─────┘
      ↓
对LL继续分解...
```

### 7.5 快速小波变换（FWT / Mallat 算法）

**分解过程：**

1. **近似系数 $cA_j$：** 低频分量\
   $cA_j[n] = \sum_{k} h[k-2n] \cdot cA_{j-1}[k]$

2. **细节系数 $cD_j$：** 高频分量\
   $cD_j[n] = \sum_{k} g[k-2n] \cdot cA_{j-1}[k]$

其中：
- $h$：低通滤波器
- $g$：高通滤波器（$g[n] = (-1)^n h[1-n]$）

**重构过程：**

$cA_{j-1}[n] = \sum_{k} h[n-2k] \cdot cA_j[k] + \sum_{k} g[n-2k] \cdot cD_j[k]$

**金字塔结构：**

```plaintext
     原始图像
         │
  ┌─────────┼─────────┐
  │         │         │
  A₁       D₁        D₁
  │         │
┌──┼──┐  ┌──┼──┐
│  │  │  │  │  │
A₂ D₂ A₂ D₂
│  │
... ...
```

### 7.6 二维小波变换的分解与重构

**分解步骤：**
1. 对每一行应用一维 DWT（低通+高通）
2. 对每一列应用一维 DWT
3. 得到 4 个子带

**重构步骤：**
1. 对每一列应用逆 DWT
2. 对每一行应用逆 DWT

### 7.7 小波包

**传统小波变换的问题：**
- 只对低频部分递归分解
- 高频部分没有被充分分析

**小波包的优势：**
- 对高频部分也进行分解
- 形成完整的二叉树结构

```plaintext
传统小波分解：        小波包分解：
    LL                  ALL
   /  \                /   \
  LL  HL  →          ALL  AHL
  │                 /  \
 ALL              ALL AHL
                  ...
```

**最优基选择：**
根据代价函数（如熵）选择最优分解路径

### 7.8 多分辨率分析

**核心思想：**
- 在不同分辨率（尺度）下观察图像
- 粗分辨率：观察整体轮廓
- 细分辨率：观察细节

**应用场景：**

| 应用 | 说明 |
| --- | --- |
| 图像融合 | 不同分辨率图像合并 |
| 图像去噪 | 小波域阈值处理 |
| 边缘检测 | 多尺度边缘检测 |
| 图像增强 | 不同尺度细节增强 |

### 7.9 小波去噪

**基本原理：**
1. 进行小波变换
2. 对细节系数进行阈值处理（软阈值/硬阈值）
3. 重构图像

**阈值处理：**
- **硬阈值：** $|w| > T \Rightarrow w$ 保留，否则设为 0
- **软阈值：** $|w| > T \Rightarrow w - T$，否则设为 0

## 第 8 章 图像压缩

### 8.1 图像压缩的基本原理

**为什么图像可以压缩？**

图像数据存在大量冗余，去掉冗余信息后仍可恢复或近似恢复图像。

**三种冗余类型：**

| 冗余类型 | 说明 | 解决方法 |
| --- | --- | --- |
| 编码冗余 | 用更多比特表示信息 | 变长编码 |
| 心理视觉冗余 | 人眼不敏感的信息 | 量化丢弃 |
| 空间冗余 | 相邻像素相似 | 预测编码/变换编码 |
| 时间冗余 | 视频帧间相似 | 帧间预测 |

**压缩性能指标：**
- **压缩比：** $CR = \frac{n_1}{n_2}$（原始比特数/压缩后比特数）
- **比特率：** 每像素比特数 bpp
- **峰值信噪比：** $PSNR = 10 \log_{10} \frac{255^2}{MSE}$

### 8.2 图像编码模型

```plaintext
原始图像 → 映射器 → 量化器 → 符号编码器 → 压缩数据
           ↑                    ↓
        映射变换              熵编码

各部分作用：
- 映射器：变换图像数据（减少空间冗余）
- 量化器：减少心理视觉冗余（有失真）
- 熵编码：减少编码冗余（无失真）
```

### 8.3 信息论基础

**信源熵（Shannon Entropy）：**

$H = -\sum_{i=0}^{L-1} p(a_i) \log_2 p(a_i)$

**物理意义：**
表示信源的平均信息量（最小平均码长）

**香农编码定理：**
- 下界：$H \leq \bar{L}$
- 存在无失真编码达到熵

**编码效率：**

$\eta = \frac{H}{\bar{L}}$

### 8.4 无失真压缩方法

#### （1）Huffman 编码

**原理：**
频率高的符号用短码，频率低的符号用长码

**步骤：**
1. 统计各符号频率
2. 构建 Huffman 树
3. 分配码字

**示例：**

```plaintext
符号:  A   B   C   D   E
频率: 15   7   7   6   5
码字: 01 110 111 100 101

平均码长 = (15+7+7+6+5)×权值 = ...
```

**特点：**
最优前缀码，但需要额外传输码表

#### （2）算术编码

**原理：**
将整个序列映射到一个实数区间

**优势：**
- 比 Huffman 编码效率更高
- 不需要传输码表
- 适合长序列

#### （3）LZW 编码（Lempel-Ziv-Welch）

**原理：**
动态建立字典，重复模式用字典索引代替

**应用：**
GIF、PNG、ZIP

**示例：**

```plaintext
原序列: ABABABAB
字典: A=1, B=2, AB=3, ABA=4, ABAB=5
编码: 12 3 4 5
```

#### （4）行程编码（RLE）

**原理：**
相同像素的连续序列用"数量+值"表示

**示例：**

```plaintext
原始: AAAAAABBBCCCDDDD
编码: 6A3B3C4D
```

**缺点：**
对随机图像效果差

#### （5）位平面编码

**原理：**
将图像分解为多个位平面，分别编码

```plaintext
8位灰度图像分解为8个位平面：
- 位平面7（最高位）：包含图像主要结构
- 位平面0（最低位）：包含细节和噪声
```

**特点：**
- 高位平面信息量大，可精细编码
- 低位平面可粗糙编码或丢弃

### 8.5 有限失真压缩

#### （1）预测编码

**原理：**
根据前面像素预测当前像素，只编码预测误差

**DPCM 系统：**

```plaintext
当前像素 → 预测器 → 预测值 ─┬─→ 减法器 → 误差
                           │     误差编码器 ──→ 传输
                           │
                           └─→ 加法器 ←─ 预测器 ←─ 前一像素
```

**预测器类型：**
- 1-D 预测：只参考左边像素
- 2-D 预测：参考左、上、左上像素

**优势：**
误差通常比原像素值小，可用更少比特表示

#### （2）变换编码

**原理：**
1. 对图像进行可逆变换（DCT、小波等）
2. 量化变换系数
3. 对量化后的系数编码

**为什么有效？**
- 变换后能量集中到少数系数
- 大部分系数接近零，便于压缩

### 8.6 JPEG 压缩（重点）

**JPEG 基本流程：**

```plaintext
原始图像
  │
  ├─ 颜色空间转换 (RGB → YCbCr)
  │
  ├─ 分块 (8×8像素块)
  │
  ├─ DCT变换 (每个块)
  │
  ├─ 量化 (频率加权)
  │
  ├─ Zigzag扫描
  │
  └─ Huffman/算术编码
```

**分块：**
- 图像划分为 8×8 块
- 减少计算复杂度
- 限制块效应在局部

**DCT 变换：**

$F(u,v) = \frac{1}{4} C(u) C(v) \sum_{x=0}^{7} \sum_{y=0}^{7} f(x,y) \cos\left[\frac{(2x+1)u\pi}{16}\right] \cos\left[\frac{(2y+1)v\pi}{16}\right]$

其中 $C(u), C(v) = \begin{cases} \frac{1}{\sqrt{2}} & u=0 \\ 1 & u\neq 0 \end{cases}$

**DCT 系数含义：**

| 位置 | 含义 |
| --- | --- |
| 左上 (0,0) | DC 系数，块的平均亮度 |
| 右上到左下 | 低频到高频细节 |

**量化表：**
- 对不同频率成分进行不同强度量化
- 低频：精细量化
- 高频：粗糙量化（可丢弃）

**标准量化表：**

```plaintext
亮度量化表：
 16  11  10  16  24  40  51  61
 12  12  14  19  26  58  60  55
 14  13  16  24  40  57  69  56
 14  17  22  29  51  87  80  62
 18  22  37  56  68 109 103  77
 24  35  55  64  81 104 113  92
 49  64  78  87 103 121 120 101
 72  92  95  98 112 100 103  99
```

**Zigzag 扫描：**
- 将 8×8 矩阵转为一维序列
- 聚集零值，便于游程编码

```plaintext
Zigzag路径：
 0  1  5  6 14 15 27 28
 2  4  7 13 16 26 29 42
 3  8 12 17 25 30 41 43
 9 11 18 24 31 40 44 53
10 19 23 32 39 45 52 54
20 22 33 38 46 51 55 60
21 34 37 47 50 56 59 61
35 36 48 49 57 58 62 63
```

**熵编码：**
- DC 系数：差分编码
- AC 系数：游程编码 + Huffman 编码

### 8.7 JPEG2000 压缩

**相比 JPEG 的改进：**

| 特性 | JPEG | JPEG2000 |
| --- | --- | --- |
| 变换方法 | DCT | DWT |
| 压缩效率 | 好 | 更好 |
| 支持渐进传输 | 否 | 是 |
| 感兴趣区域（ROI） | 否 | 是 |
| 抗误码能力 | 差 | 好 |
| 有损/无损 | 仅支持有损 | 同时支持 |

**JPEG2000 基本流程：**

```plaintext
原始图像
  │
  ├─ 分块 (更大，如256×256)
  │
  ├─ 小波变换
  │
  ├─ 量化
  │
  └─ 熵编码 (EBCOT)
```

**EBCOT 编码：**
- 嵌入式编码
- 比特平面编码
- 支持质量渐进

### 8.8 视频压缩基础

**帧类型：**

| 帧类型 | 说明 | 压缩方式 |
| --- | --- | --- |
| I 帧 | 完整帧，可独立解码 | 帧内 JPEG |
| P 帧 | 前向预测帧 | 参考前 I/P 帧 |
| B 帧 | 双向预测帧 | 参考前后 I/P 帧 |

**运动估计：**
- 在参考帧中搜索当前块的最佳匹配
- 只编码运动矢量 + 预测误差

**常见标准：**

| 标准 | 特点 |
| --- | --- |
| MPEG-1 | VCD 质量 |
| MPEG-2 | DVD 质量 |
| MPEG-4 | 面向对象 |
| H.264/AVC | 高压缩比，广泛应用 |
| HEVC/H.265 | 比 H.264 好 50%+，4K/8K |

### 8.9 压缩标准总结

| 标准 | 变换方法 | 有损/无损 | 主要应用 |
| --- | --- | --- | --- |
| JPEG | DCT | 有损 | 照片、网页 |
| JPEG2000 | DWT | 两者 | 医学、遥感 |
| GIF | LZW | 无损 | 简单动画 |
| PNG | LZ77+哈夫曼 | 无损 | 需要透明 |
| TIFF | 多种 | 两者 | 专业图像 |
| WebP | DCT+VP8 | 两者 | 网页 |
| AVIF | HEIF | 有损 | 高效网页 |
